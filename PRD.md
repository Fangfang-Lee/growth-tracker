# Growth Tracker 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品名称
**Growth Tracker** - 个人成长追踪工具

### 1.2 产品定位
一款帮助用户规划、追踪和复盘个人成长计划的工具，通过AI Agent分析用户习惯，自动生成个性化的计划提醒。

### 1.3 核心价值
- **循环计划**：一次创建，自动每周生成计划实例
- **灵活打卡**：支持进度型（百分比）和次数型（可量化）两种打卡方式
- **习惯洞察**：AI分析用户最佳完成时间，固化习惯
- **进度可视化**：直观展示成长轨迹

### 1.4 目标用户
- 想要系统性提升自己的职场人士/开发者
- 年龄段：22-35岁
- 使用场景：自我提升、求职准备、技能学习

---

## 2. 核心概念

### 2.1 循环计划
循环计划是用户创建的"计划模板"，系统会每周自动生成一个计划实例。

```
示例：
用户创建 "每周锻炼4次"
→ 系统每周一自动生成当周的4次锻炼机会
→ 用户每次完成点击一下
→ 无需每周手动创建
```

> 周起始日：默认周一至周日，可在设置中配置（可改为周日到周六）。

### 2.2 打卡类型

| 类型 | 说明 | 示例 | 打卡方式 |
|------|------|------|----------|
| **进度型** | 需要多天完成，用户自行评估进度 | AI工具解析 | 每天更新进度：30%→50%→100% |
| **次数型** | 可量化，每次完成点击一下 | 锻炼、算法题、博客 | 每次完成点击一下 |

### 2.3 分类
分类由用户自行创建，不再预设。每个分类只定义打卡类型（进度型/次数型），不设置具体目标。

```
用户创建的分类示例：
├── AI学习（进度型）
├── 技术博客（次数型）
├── 生活博客（次数型）
├── 算法训练（次数型）
├── 锻炼（次数型）
└── 系统设计（次数型）
```

> 注意：目标只在循环计划层设置，分类仅负责定义打卡类型。

## 5. Agent 智能体设计

### 5.1 习惯分析

Agent 自动收集用户的打卡时间，统计分析形成习惯报告。

| 分析维度 | 说明 |
|----------|------|
| 最佳完成日 | 周几完成最多 |
| 最佳完成时段 | 早上/下午/晚上/具体时间 |
| 完成频率稳定性 | 波动大还是稳定 |
| 遗漏模式 | 哪几天最容易忘记 |

### 5.2 习惯洞察示例

```
🤖 习惯洞察

📅 算法训练
   最佳完成日: 周一至周五
   最佳时段: 21:00-22:00
   置信度: 85%
   → 建议: 保持现有习惯，晚间专注力好

🏃 锻炼
   最佳完成日: 周三、周六
   最佳时段: 上午 (7:00-9:00)
   置信度: 60%
   → 建议: 可尝试周日上午，已有过成功案例

📝 技术博客
   最佳完成日: 周六
   最佳时段: 14:00-18:00
   置信度: 75%
   → 建议: 周六下午预留3-4小时专注写作
```

### 5.3 智能提醒

| 触发条件 | 提醒内容 |
|----------|----------|
| 计划设定时间前15分钟 | "15分钟后是你计划的算法训练时间" |
| 某计划连续2天未完成 | "你已经连续2天没打卡算法训练了" |
| 周末计划完成率低 | "本周还有3项计划未完成，周日加油！" |

## 8. 数据模型

```
用户 (users)
```
